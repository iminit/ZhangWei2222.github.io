vuex源码分析(原理与安装)

发表于 2019-12-03
更新于 2020-02-04
分类于 源码分析 阅读次数：

原理

每个组件（也就是Vue实例）在beforeCreate的生命周期中都混入（Vue.mixin）同一个Store实例 作为属性 $store，所以我们能够使用this.$store；

里面有一个resetStoreVM()方法，通过vm重设store，新建Vue对象使用Vue内部的响应式实现注册state以及computed，我们在获取state的时候，拿的是vm.$_store._vm._data.$$state，这个变量是响应式的

Vue.use 安装

因为Vue使用插件时，插件必须提供一个install方法

install 函数

vuex/src/store.js

  1. 防止Vuex被重复安装
  2. 执行applyMixin，目的是执行vuexInit方法初始化Vuex
    • 如果是Vue1.0，Vuex会将vuexInit方法放入Vue的_init方法中
    • 如果是Vue2.0，则会将vuexinit混淆进Vue的beforeCreacte钩子中

applyMixin 函数

vuex/src/mixin.js

在vue的声明周期中进行vuex的初始化，并且对vue的各种版本进行了兼容。vuexInit就是对vuex的初始化。为什么我们能在使用vue.$store这种方法呢，因为在vuexInit中为vue的实例添加了$store属性

这里的store就是Vuex.Store，我们通常在store/index.js里面声明的：

# Vuex # Vue
